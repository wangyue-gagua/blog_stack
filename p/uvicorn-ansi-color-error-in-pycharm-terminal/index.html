Uvicorn ANSI Color Error In Pycharm Terminal

entangled with Uvicorn ANSI escape color display error on Pycharm

May 11, 2021

Origin

Recently I’m learning FastAPI wanting to develop a JSON API based backend server which using Uvicorn as ASGI server. Previously I usually used VSCode to writing python script, but this time I’d like to build a slightly bigger project so I turned to Pycharm to build FastAPI application.

Uvicorn

Uvicorn is a lightning-fast ASGI server implementation, using uvloop and httptools.

ERROR

When I ran FastAPI application with uvicorn.exe sql_app.main:app I got into a weird trouble:

Uvicorn’s Issue

After Googled for this kind of strange behavior, I found an issue opened on FastAPI page which is related to uvicorn’s bug.

That bug seems caused by color escaping error on windows, and euri10 created a pull request which add click.clear() on logging.py to fix the bug. However, this line disappeared at official Uvicorn repository and the package I downloaded. Despite it actually works, I think modify the package hardcode by myself is not an elegant solution.

ANSI Escape Code

After spending a lot of time searching for that issue, I finally got some clues about it.

ANSI escape sequences are a standard for in-band signaling to control cursor location, color, font styling, and other options on video text terminals and terminal emulators. [32mINFO[0m: is equivalent to \e[32mINFO\e[0m which will display INFO on terminal. \e means escaping, [32m means green foreground and [0m reset the escape color code.

Windows Terminal

Installed Windows terminal at Microsoft store, then I tried launching Uvicorn with windows terminal. To my surprise, both powershell and gitbash rendered colorful Uvicorn’s starting log on terminal.

Strange Behavior

then I tried running echo -e "\e[37;44;3;1mLYL\e[0m" on different terminal with different shell, this code expected white italic “LYL” text with blue background looks like: LYL

On windows terminal platform print with correct format:

but bash.exe can’t render italic text:

git-bash.exe have completely correct format, and run with --use-colors parameter has colorful log output:

And I tried setting Pycharm terminal’s shell as git-bash.exe which will have wonderful formatted log, with only one drawback: a seperate window.

Conclusion

Different terminals or terminal emulators have different behaviors, that makes me frustrating, I finally couldn’t find a perfect solution about this problem.

Pycharm integrated terminal as an embedded terminal emulator, it has it’s own exclusive manifestation. As a non-ANSI-aware terminal it has intrinsic flaw. VsCode might be a better solution?
